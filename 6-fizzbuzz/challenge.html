Challenge: FizzBuzz

The young campers are playing a game where they count from 1 to 100 one number at a time. For each number:

  • If the number is divisible by 3, they say "Fizz"
  • If the number is divisible by 5, they say "Buzz"
  • If the number is divisible by 3 and 5, they say "FizzBuzz"
  • If the number is not divisible by 3 or 5, they say the number

For instance, here's how a game might go:

      1, 2, Fizz, 4, Buzz, 5, Fizz, 7, 8, Fizz, Buzz, 11, Fizz, 13, 14, FizzBuzz, 16, 17, Fizz ... 97, 98, Fizz, Buzz

Implement a program that simulates a game of FizzBuzz using JavaScript

Hints

  • What programming construct can be used to count up and execute code many times?
  • What operator returns the remainder of a division operation? How do you use it to check divisibility?

Take It Further

The campers get bored easily, so they want to mix up the rules a bit. They might:

  • Change the number they start counting at
  • Change the number they count up to
  • Change the amount of numbers they count up each turn
  • Add or remove divisibility rules
    • Remove 3 and "Fizz"
    • Add 7 and "Bark"

You don't want to make manual modifications to the code every time the rules change. How would you update your code to handle these conditions for future runs?

Hint: how could you utilize function parameters and/or an object literal to make this possible?
